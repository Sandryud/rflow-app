# Модуль `approvals`

## Назначение

Модуль отвечает за approval-процесс релиза.

Его основная роль:

- создавать approval для конкретного reviewer и релиза;
- возвращать список approval-запросов релиза;
- фиксировать решение reviewer в статусах `PENDING`, `APPROVED` и `REJECTED`;
- отзывать принятое решение обратно в `PENDING`;
- удалять назначение reviewer, пока релиз находится в `DRAFT`;
- поддерживать процесс проверки release до перехода в следующий статус workflow.

Модуль работает в контексте релиза и организации. Он не описывает бизнес-логику публикации релиза напрямую, а только approval-сценарии для неё.

Подробные сценарии HTTP-операций описаны в отдельных endpoint-документах в [src/modules/approvals/docs/endpoints](./endpoints).
