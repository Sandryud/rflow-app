# Модуль `releases`

## Назначение

Модуль отвечает за релизы проекта.

Его основная роль:

- создавать и хранить релизные сущности в рамках проекта;
- возвращать релиз по идентификатору и список релизов проекта;
- поддерживать релизные задачи, которые входят в конкретный релиз;
- фиксировать статус релиза и запрещать добавление задач, если релиз уже не находится в `DRAFT`.

Модуль работает в контексте проекта и организации, но не управляет самим деплоем или окружениями как таковыми.

Подробные сценарии HTTP-операций описаны в отдельных endpoint-документах в [src/modules/releases/docs/endpoints](./endpoints).
