# Модуль `comments`

## Назначение

Модуль отвечает за обсуждение релиза участниками организации и хранение комментариев с неизменяемым авторством.

## Основные обязанности

- создавать, возвращать, изменять и мягко удалять `Comment`;
- управлять связями `Release → Comment` и `User → Comment(author)`;
- проверять membership, ownership и право модерации;
- исключать soft-deleted comments и недоступную родительскую цепочку;
- сохранять структуру mutation-операций для будущего audit trail.

## Что не относится к этому модулю

- управление статусами и workflow релиза;
- управление memberships и ролями организации;
- создание `AuditEvent` и notifications;
- изменение данных автора комментария.

## Контекст использования

Модуль участвует в обсуждении релиза на всех этапах его жизненного цикла. Он использует release и organization context для проверки доступа, а данные автора получает из JWT и связи с `User`.
